# Histórico de Setup e Comandos do Projeto


Meu mini dicionario Git:

feat: (feature) -> Quando você cria uma funcionalidade nova 
fix: -> Quando você está corrigindo um bug.
docs: -> Quando você atualiza a documentação
chore: -> Quando você faz coisas de manutenção 


1. Inicialização do projeto e configuração do ambiente TypeScript:
npm init -y
npm install -D typescript @types/node tsx
npx tsc --init

2. Execução do script inicial:
npx tsx --watch src/index.ts

3. Configuração de live-reload para o desenvolvimento:
npm install -D nodemon ts-node
- O nodemon reinicia a aplicação a cada alteração salva.
- O ts-node é necessário para o nodemon compilar e rodar o código em TypeScript nativamente.

4. Instalação de dependências para o web scraping:
npm install axios
- O axios é responsável por fazer as requisições HTTP para a URL alvo (baixar o conteúdo HTML).

5. Controle de versão com Git:
- Repositório criado e linkado ao GitHub.
- Configuração do arquivo .gitignore para não versionar arquivos pesados (node_modules), dados locais gerados (JSON/CSV) .
- Realizado o primeiro commit com a versão funcional do scraper.

 6. Qualidade de Código (Linter e Testes Automáticos):
 - Instalação do ESLint para garantir os padrões do TypeScript e evitar erros de sintaxe.
 - Instalação do Jest e ts-jest para criação de testes automatizados.
 - Criados os arquivos de configuração (eslint.config.js e jest.config.cjs).
 - Adicionados os comandos "lint" e "test" no package.json.
 - Criado um teste inicial de exemplo para validar o pipeline de CI/CD.

7. Containerização (Docker):
- Criado o arquivo Dockerfile utilizando Multi-stage build para separar o ambiente de instalação do ambiente de execução final.
- Configurada a imagem base node:20-alpine por ser extremamente leve.
- Adicionada a instrução USER node para garantir a execução do container sem privilégios de root (boas práticas de segurança).
- Criado o arquivo docker-compose.yml (Bônus do desafio) para rodar a aplicação localmente e mapear o volume, permitindo que os arquivos gerados (CSV/JSON) apareçam na máquina local.
